Building source code of LibTiff.Net

LibTiff.Net, the managed version of the LibTiff library for reading and writing TIFF.

This article explains how to build source code package of LibTiff.Net library.

If you prefer to use pre-built LibTiff.Net library, please read this article. It describes the steps you need to take to setup your environment to read and write TIFF files in your applications.

The source code of the library may be built using Microsoft Visual Studio 2005 or later (general .NET version). Silverlight version of the library may be built using Microsoft Visual Studio 2008 or later.

 Note

The zip with latest source code release of the LibTiff.Net library can always be downloaded from http://bitmiracle.com/libtiff.net_src.zip.

 Note

The source code package of the LibTiff.Net library contains source code for the library, all tools and sample applications. Unit tests and test images are also included in source code package.

Building LibTiff.net using Microsoft Visual Studio 2005

First of all, you need to extract source code package into a folder of your choice.

Next, open the folder that you extracted source code package to. To build source code of the library just open LibTiff.NET.2005.sln file from the folder in Microsoft Visual Studio 2005 and build the solution.

Building LibTiff.net using Microsoft Visual Studio 2008

First of all, you need to extract source code package into a folder of your choice.

Next, open the folder that you extracted source code package to. You have two choices:

  • To build just general .NET version of the library open LibTiff.NET.2008_NoSilverlight.sln file from the folder in Microsoft Visual Studio 2008 and build the solution.
  • To build general .NET version and Silverlight version of the library open LibTiff.NET.2008.sln file from the folder in Microsoft Visual Studio 2008 and build the solution.

 Note

To build Silverlight version of the LibTiff.Net using Microsoft Visual Studio 2008 you'll need Microsoft Silverlight 3 SDK and Microsoft Silverlight 3 Tools for Visual Studio 2008 SP1 installed.

Building LibTiff.net using Microsoft Visual Studio 2010

First of all, you need to extract source code package into a folder of your choice.

Next, open the folder that you extracted source code package to. You have three choices:

  • To build just general .NET version of the library open LibTiff.NET_NoSilverlight file from the folder in Microsoft Visual Studio 2010 and build the solution.
  • To build general .NET version and Silverlight version of the library targeting Silverlight 3 runtime open LibTiff.NET_Silverlight3 file from the folder in Microsoft Visual Studio 2010 and build the solution.
  • To build general .NET version and Silverlight version of the library targeting Silverlight 4 runtime open LibTiff.NET file from the folder in Microsoft Visual Studio 2010 and build the solution.

 Note

To build Silverlight version of the LibTiff.Net targeting Silverlight 3 runtime using Microsoft Visual Studio 2010 you'll need Microsoft Silverlight 3 SDK installed.

 Note

To build Silverlight version of the LibTiff.Net targeting Silverlight 4 runtime using Microsoft Visual Studio 2010 you'll need Microsoft Silverlight 4 SDK and Microsoft Silverlight 4 Tools for Visual Studio 2010 installed.

Building documentation for LibTiff.net (Optional)

First of all, you need to extract source code package into a folder of your choice.

Next, open the folder that you extracted source code package to. After that open LibTiff subfolder in that folder. To build documentation for LibTiff.Net just open Help.shfbproj file from the folder in Sandcastle Help File Builder GUI and build the project.

 Note

To build documentation for the LibTiff.Net you'll need latest Sandcastle (June 2010 Release), Sandcastle Help File Builder GUI (v1.9.1.0), and Sandcastle June 2010 Patch - Rev 1 patch from Sandcastle Styles Project installed.
